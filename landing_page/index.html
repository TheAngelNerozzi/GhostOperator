Common Questions

Does GhostOperator send data to the cloud?
No. GhostOperator is strictly local-first. Mission logic, vision processing, and automation protocols are executed entirely on your machine via 127.0.0.1.
What are the minimum hardware requirements?
GhostOperator adapts to any hardware. While a GPU (NVIDIA 8GB+ VRAM) is recommended for sub-2s response, the 'Adaptive Fallback' mode supports dual-core CPUs with 8GB RAM.
Can it bypass anti-bot detections?
Yes. The Ghost Glide™ algorithm uses organic easing and noise interpolation for mouse movements, making action execution look like natural human input.

02 Custom Recipes (.yaml)

Recipes are the automation blueprint for GhostOperator. They define sequential goals, conditions, and vision-logic parameters in a human-readable YAML format.

example_recipe.yaml
version: "1.0"
name: "Firefox Social Bot"
steps:
  - goal: "Open Firefox and go to github.com"
    max_retries: 3
    timeout_ms: 15000
  
  - goal: "Find the Login button and click it"
    vision_density: "20x20"
    action: "CLICK"

  - goal: "Enter credentials and submit"
    condition: "ELEMENT_EXISTS(input#login)"

Structure Rules

  • Must include a unique `name`.
  • `steps` is an ordered list of tasks.
  • `goal` uses Natural Language reasoning.

Pro Parameters

  • `vision_density`: Control grid resolution.
  • `fallback_budget`: ms for weak hardware.
  • `stealth_mode`: Organic click jitter.

Complete Running Documentation

A step-by-step guide to deploying GhostOperator in your local environment.

Step 0: Option A - Pro Docker Cloud

The cleanest way to deploy the GhostOperator brain on any OS.

# Pull and run directly from root
docker build -t ghost-operator .
docker run -p 7474:7474 ghost-operator

Step 0: Option B - Native Setup

  • ▹ Golang 1.22+ installed and configured.
  • ▹ Ollama running locally.

Step 1: Environment Setup

Navigate to the root project folder and ensure your local LLM is reachable.

# Start Ollama service
ollama serve

# Pull the vision model
ollama pull moondream

Step 2: Start UI & Engine

Run the unified setup script which compiles the vision-orchestrator and boots the HTTP server.

# Execute installer
./setup.sh

Step 3: Access Dashboard

GhostOperator exposes a modern real-time dashboard on your local loopback.
